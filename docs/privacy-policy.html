SnipKit / Privacy Policy

Privacy Policy

Last updated: April 1, 2026  ·  SnipKit v1.0.0

Overview

SnipKit is a text expansion tool that runs entirely inside your browser. It does not collect, transmit, or store any information outside your device. There are no servers, no accounts, no analytics, and no external network requests of any kind. What you type stays on your machine.

Data We Collect

None. SnipKit does not collect any user data.

No usage statistics, no crash reports, no telemetry, no identifiers, no IP addresses. The extension has no mechanism to send any information anywhere — because it has no backend to send it to.

Data Storage

Your snippets (triggers and their expansion text) are stored in chrome.storage.local — the browser's built-in local storage API. This data lives exclusively on your device inside your Chrome profile.

This data is never transmitted to any server. It is never synced to Google's cloud (we do not use chrome.storage.sync). It is never shared with any third party.

If you uninstall SnipKit, your snippet data remains in your browser's local storage until Chrome clears it as part of its normal profile management.

Permissions Explained

storage Allows the extension to save your snippet library to chrome.storage.local on your device. Without this, snippets could not persist between sessions.
activeTab Allows the extension to interact with the currently active browser tab. This is required to detect text fields, inject expanded text, and open the command palette. SnipKit only reads from and writes to the focused input element — it does not read page content, form data, or any other information from the tab.
tabs Required to send the "open palette" message from the background service worker to the content script running in the active tab when you press Ctrl+Shift+Space. No tab URLs or tab content are read.
clipboardWrite Used by the command palette to copy your selected snippet to the clipboard so it can be pasted into canvas-based editors like Google Docs, where direct DOM injection is not possible.
clipboardRead Declared alongside clipboardWrite as required by Chrome's clipboard permission model. SnipKit does not read clipboard contents.

Third Parties

SnipKit does not integrate with any third-party services. There are no analytics providers (no Google Analytics, no Mixpanel, no Sentry). There are no advertising networks. There are no feature flags or remote configuration services.

The extension makes zero outbound network requests at runtime. You can verify this by opening Chrome DevTools → Network tab while SnipKit is running — it will be empty.

Children's Privacy

SnipKit does not collect any data from any users, including children under 13. There is nothing to collect.

Changes to This Policy

If this privacy policy changes in a future version, the updated policy will be published at this URL and the "Last updated" date at the top of this page will be revised. Material changes that affect data handling will also be noted in the extension's Chrome Web Store changelog.
